Before we start:

You can follow this guide on an already running system or make a fresh installation.

If you decide for a clean new server, just follow the official documentation

Attention:

openSUSE is using firewalld and all ports are blocked by default. This means that your machine is not able to send data via ports per default.

Here are more optional information about this topic.

There are two option to avoid problems.

Option 1:

  1. If you make a fresh installation of openSUSE leap 15.3, you can just turn off the firewall on the last step of the installation process.

    In this way, you ensure, that your fresh system firewall is disabled, and you can access it from another computer via ssh.

Option 2:

  1. Stop the firewall on your running system
    sudo systemctl stop firewalld.service
  1. Disable the firewall, so on the next reboot it doesn't start the firewall again
    sudo systemctl disable firewalld.service

  1. Check if the firewall service has stopped 
    systemctl is-active firewalld

    Expected Output: inactive

  1. Check if the firewall is disabled
    systemctl is-enabled firewalld

    Expected Output: disabled

Now you are good to go and can start with the How-to guide.

  1. Per default, docker is not on after installation. You need to start the software.

STEP 2 Run Nextcloud docker container and make it available on the home network

  1. Open up a new terminal like in STEP 1 or just use the already opened terminal
  1. Now we need to create a working directory for our upcoming Nextcloud docker container Let's create a docker directory which includes a Nextcloud directory in your home directory like this.
    mkdir ~/docker/nextcloud
  1. Move inside your newly created working directory
    cd ~/docker/nextcloud
  1. Now we need to create a configuration file for our docker container, which we can also edit

    With vim, we will create a new empty file called docker-compose.yml. This file will contain all our configuration for the docker container.

    vim docker-compose.yml
  1. Fill content of docker-compose
    version: "3"

volumes:
  nextcloud:
    driver: local
  db:

networks:
  nc-nginx:

services:
  db:
    image: mariadb:10.5
    restart: always
    command: --transaction-isolation=READ-COMMITTED --binlog-format=ROW
    volumes:
      - db:/var/lib/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=<INSERT-YOUR-PASSWORD>
      - MYSQL_PASSWORD=<INSERT-YOUR-PASSWORD>
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
    networks:
      nc-nginx:

  app:
    image: nextcloud
    restart: always
    ports:
      - 8080:80
    links:
      - db
    volumes:
      - nextcloud:/var/www/html
    environment:
      - MYSQL_PASSWORD=<INSERT-YOUR-PASSWORD>
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_HOST=db
    networks:
      nc-nginx:

Summary

If everything worked out, you should own now your personal own cloud server. You can access it via a subdomain delivered by Duck DNS. A bash script will update your IP every 5 minutes and ensure your server is reachable even if you don't have a static IP.

The Nginx Proxy Manager will redirect your, encrypted data to your Nextcloud docker container.

Nextcloud is your personal cloud, and you are again in the control of your data.

Congratulation, you did it!
